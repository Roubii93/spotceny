Node-RED na Cerbo GX — automatizace ESS bez programování

M
Michal
· Únor 2026 · 9 min čtení

Victron Cerbo GX má vestavěný Node-RED. Většina majitelů o tom neví, nebo to považuje za „složité pro programátory". Není to pravda. Node-RED je vizuální nástroj — spojujete bloky šipkami. Za odpoledne máte automatizaci, která funguje lépe než jakékoli manuální přepínání.

Co je Node-RED a proč na Cerbo GX?

Node-RED je open-source nástroj od IBM pro vizuální programování. Namísto psaní kódu táhnete a propojujete bloky (nodes). Cerbo GX — řídicí počítač systému Victron — ho má předinstalovaný od verze firmware 2.80.

Proč Cerbo GX + Node-RED?

Cerbo GX má přímý přístup k Victron D-Bus — systémové sběrnici kde jsou všechna data: SoC baterie, výkon FVE, síťový výkon, nastavení ESS. Node-RED umí tato data číst a zapisovat. Tím získáte plnou kontrolu nad systémem bez závislosti na cloudu nebo externích API.

Jak zprovoznit Node-RED na Cerbo GX

1

Aktualizujte firmware na 2.80+

V VRM portálu nebo přímo na Cerbo GX (Settings → Firmware → Online updates). Node-RED je dostupný od verze 2.80.

2

Zapněte Node-RED v nastavení

Settings → Venus OS Large features → Node-RED: ON. Cerbo GX se restartuje.

3

Otevřete Node-RED editor

V prohlížeči přejděte na http://[IP-adresa-Cerbo]:1880. IP adresu najdete v Settings → Ethernet nebo WiFi. Editor se otevře v prohlížeči.

4

Nainstalujte node-red-contrib-victron

Menu (hamburger) → Manage palette → Install → hledat node-red-contrib-victron. Tato knihovna přidá speciální nodes pro čtení a zápis D-Bus hodnot.

Naše základní flow: optimalizace podle spotových cen

Flow funguje takto: každou hodinu se zkontroluje aktuální spotová cena z naší API, porovná se SoC baterie a na základě toho se nastaví ESS mód (nabíjet ze sítě / vybíjet do sítě / normální provoz).

Logika flow

Pokud spotová cena < 40 EUR/MWh AND SoC < 85 %: Zapni nabíjení ze sítě (ESS mód: Keep batteries charged)

Pokud spotová cena > 120 EUR/MWh AND SoC > 30 %: Vybíj baterii (ESS mód: Optimized with BatteryLife)

Jinak: Normální provoz — baterie chrání dům od sítě

Základní HTTP request node pro stažení cen

// Function node: připraví URL pro aktuální hodinu const now = new Date(); const hour = now.getHours(); const url = `https://mh-energy.vercel.app/api/eu-prices?country=cz`; msg.url = url; msg.payload = { hour }; return msg;

Za HTTP request node přidejte Function node, který zpracuje odpověď a najde cenu pro aktuální hodinu. Pak Switch node, který rozdělí tok podle prahů ceny. Na konci každé větve je Victron write node, který zapíše nastavení ESS.

Přebytky FVE → nabíjení elektromobilu

Druhé flow sleduje výkon FVE a SoC baterie. Když baterie přesáhne 90 % a FVE stále vyrábí, pošle příkaz wallboxu přes EVCC, aby začal nabíjet auto z přebytků.

Požadavky

Pro toto flow potřebujete wallbox kompatibilní s EVCC (seznam na evcc.io). EVCC musí běžet na síti — ideálně na Raspberry Pi nebo NAS. Node-RED pak komunikuje s EVCC přes jeho REST API.

Kolik nás stálo nastavení oproti manuálu?

Než jsme měli Node-RED flow, přepínali jsme ESS mód ručně přes VRM portál — průměrně 3–4× denně. To zabralo asi 10 minut denně = 60 hodin ročně. A stejně jsme reagovali se zpožděním.

Nastavení Node-RED flow trvalo jedno odpoledne (cca 4 hodiny) včetně testování. Výsledek: automatizace reaguje každou hodinu, my se o nic nestaráme a úspora je vyšší než při manuálním řízení.
